# フレーズ暗記に挫折した私が、日本語2行で学ぶ英語アプリを開発した理由

## リード文

「この表現、どこかで見たような気がするけど…」

英語参考書のフレーズを丸暗記しても、実際の会話ではうまく使えない。参考書的な例文は記憶に残らない。自分で英文を書こうとしても、白紙からは何も出ない──。

こうした悩みを持つ学習者は、想像以上に多いと思います。読める・聞けるのに話せない、TOEIC 600～900レベルの中級者たちです。

私がこのアプリを開発した理由も、まさにここにあります。従来の「暗記→実践」という学習モデルではなく、**「自分の体験→AI生成→学習→復習」という新しい循環**を作りたかった。

このアプリが、どんな課題を解き、どんな工夫で実現しているのか、詳しく紹介していきます。

---

## セクション2：課題の深掘り「なぜ中級者は『話せない』のか」

### なぜ、高スコアでも英語が話せないのか

多くの人が経験していることですが、**TOEIC高得点＝英語が話せる、ではありません。**

その理由は、学習環境にあります。

### 問題1：インプット偏重の教育構造

学校教育から英会話スクールまで、大半の学習メディアは「読む」「聞く」に特化しています。

- 高校までの英語授業は、文法と読解が中心
- 大学入試も、読み・聞き・選択肢から選ぶ形式
- 資格試験（TOEIC など）も、リスニングと読解のみ

これらのテストで高得点を取ることは、確かに重要な基礎です。でも、**実際に口を開いて話す経験は、ほぼゼロ。**

結果として、知識はあっても「使い方」を知らない状態になってしまう。これが、中級者がぶつかる最初の壁です。

### 問題2：フレーズ暗記の限界

「中級者向け」という参考書のフレーズを覚えても、実際の会話では活躍しません。

なぜか？

参考書の例文は、あらゆるシーンに対応するように作られた**汎用的すぎる文脈**だからです。

- 「I appreciate your cooperation.」（ご協力感謝します）
- 「It would be helpful if you could…」（～していただけると幸いです）

これらは、確かに「正しい英語」です。でも、いつ実際の会話で使うのか、イメージが沸きません。そして、記憶にも残りにくい。

試しに、2週間前に覚えたフレーズを思い出そうとしてください。多くの場合、「あ、なんだっけ」という状態ではないでしょうか。

### 問題3：日記・ライティング学習の高すぎるハードル

「それなら、日記を書いて練習しよう」と考える人も多いでしょう。その発想は正しい。でも、実行は難しい。

理由は3つ：

1. **白紙恐怖症**
   - 一文目から自分で作文する必要があり、ハードルが高い
   - 「これで合ってるの？」という不安が常につきまとう

2. **添削がない**
   - ChatGPT を使えば添削は可能ですが、毎回プロンプトを入力するのは面倒
   - 「これでいい」という確信が持てない

3. **継続できない**
   - モチベーション維持が難しい
   - 1週間続いて、その後フェードアウト…という経験、ありませんか？

### 問題4：既存ツールの不便さ

では、既存の学習ツールは、この課題を解いているのか？

- **Anki（単語帳アプリ）**
  - 自分でフラッシュカードを作成する手間が膨大
  - 挫折率が高い

- **ChatGPT（生成AI）**
  - 毎回、学習用にプロンプトを作成し直す必要がある
  - 学習の履歴が残らず、復習しづらい

- **英会話アプリ**
  - 月数千円の月額費用が必要
  - 「今日は疲れているから」と言い訳しやすい

**つまり、「最小限の入力で、継続できて、かつ実用的な学習ができる」ツールが、存在しないのです。**

### まとめ：中級者が求めるもの

もう一度、整理してみます。TOEIC 600～900の中級者が、今必要としているのは：

✅ **低いハードル** - 日本語で書いて、AI がサポート
✅ **実用的な素材** - 自分の体験に基づいた、本当に使う表現
✅ **確実な記憶** - 単語帳ではなく、文脈で覚える
✅ **継続の仕組み** - 強制されるのではなく、習慣化する設計

## セクション3：解決策のコンセプト「このアプリが目指したもの」

### コアコンセプト：「日本語で2行。あとはAIにおまかせ。」

セクション2で見たような課題を前にして、「では、どう解決するか？」という問いに対する答えが、**このアプリの設計思想のすべて**に詰まっています。

シンプルに言えば：

> **最小限の入力（日本語2行）で、AIが自然な英文と、そこから抽出された学習素材を自動生成する。**

これだけです。でも、この一文の中には、3つの重要な設計思想が隠れています。

### 設計思想1：文脈ベースの学習

従来のフレーズ暗記は、**記号的で、感情的記憶につながりません。**

例えば、参考書で「I appreciate your cooperation.」を覚えたとしても：

- どういう状況で使うのか、イメージが薄い
- 自分の人生に関係がない
- だから、記憶に残りにくい

一方、もし**あなたが昨日の経験**を英語で書いたとしたら？

> 「今朝、同僚がプレゼンの手伝いをしてくれた。感謝を伝えたいけど、英語でどう言うんだろう？」

このような個人的な文脈があれば、生成された英文や表現は「自分ごと」として脳に刻まれます。

**脳は、汎用的な知識よりも、文脈的で感情的な情報をはるかによく記憶する。**

このアプリは、その原理を活用しています。

### 設計思想2：アウトプット重視

セクション2で指摘した通り、中級者に欠けているのは **「使う経験」** です。

だから、このアプリの流れは：

1. **書く**（日本語で状況を説明）
2. **確認する**（AIが生成した英文を読む）
3. **思い出す**（復習時にリコールテストで再現）
4. **声に出す**（TTS機能で音読）

つまり、**読む・聞く** だけでなく、**自分の言葉で再現する** という、アウトプット経験が組み込まれています。

第二言語習得研究でも、**アウトプット機会がある学習者ほど、実用的な言語スキルが身につく**ことが示されています。（Swain の Comprehensible Output 仮説）

### 設計思想3：継続の仕組み化

ここが、最も重要な部分かもしれません。

いくら素晴らしい学習法でも、**続かなければ意味がない。**

従来の英語学習は、「続くかどうか」を学習者の意志力に任せてしまっています。だから、多くの人が挫折する。

このアプリは、そうではなく、**継続を仕組みで支える**設計になっています：

#### ① 1日1エントリー制限で習慣化

毎日、1つだけ日記が書ける。それ以上は書けない。

これが、実は重要です。なぜなら：

- 「今日も書こう」という小さな習慣ループができる
- 「何をしよう」と迷わない（1つだけだから）
- 毎日継続することで、確実に英語に触れる時間が増える

#### ② SRS（Spaced Repetition System）で忘れた頃に復習

書いた日記の表現を、SRS という仕組みで段階的に復習します。

- 1日後
- 3日後
- 7日後
- 30日後

それぞれのタイミングで、「このフレーズ、覚えてる？」という復習がやってくる。

**忘れたくらいのタイミングで復習すると、最も記憶が定着します。** これは、Ebbinghaus の忘却曲線という心理学の古典的研究で知られている原理です。

#### ③ カレンダー・ストリーク機能で可視化

過去の日記がカレンダーに並ぶ。「○年前の今日」が見える。

これが、地味だけど、実は強力です。なぜなら：

- 自分の成長が目に見える
- 「今日も書く」というモチベーションが生まれやすい
- ストリーク（連続性）という心理的効果が働く

### 3つの思想を統合する

この3つの設計思想は、バラバラに存在しているのではなく、**1つの学習循環を作っています：**

```
あなたの体験（日本語）
    ↓
AIが英文を生成
    ↓
文脈に紐づいた学習素材
    ↓
毎日、小さく継続
    ↓
SRSで段階的に復習
    ↓
「あ、この表現、前にやった！」という記憶の再活性化
    ↓
実際の会話で、無意識に言える英語へ
```

このアプリが目指しているのは、**「勉強している感覚なく、自然に英語が身につく」という理想像**です。

それは、参考書も、月額課金も、時間拘束もない。ただ、毎日2-3分、自分の日本語を書くだけ。あとはすべてAIと仕組みが面倒を見てくれる。

### なぜ、このアプローチが有効なのか

学習心理学の観点から、この設計には根拠があります：

- **文脈学習** → Constructivism（構成主義）に基づいた効果的な学習
- **アウトプット** → Swain のComprehensible Output 仮説
- **継続** → Habit Loop（習慣の形成）
- **SRS** → Ebbinghaus の忘却曲線を応用した最適な復習タイミング

つまり、このアプリは、単なる「便利なツール」ではなく、**教育学的な根拠を持った学習システム**なのです。

## セクション4：アプリの主要機能紹介（実例ベース）

ここからは、抽象的な設計思想から、実際の使用シーンへ入ります。

具体的な例を通じて、「どうやって使うのか」がイメージできるようにしていきます。

### 4-1. ウィザード形式の簡単入力

**朝、仕事に行く前の10分。あなたはアプリを開きます。**

まず最初に見えるのが、3つのカテゴリー選択画面です：

- 📝 **今日の出来事** - 日中に起きたことを記録
- 💭 **考えたこと・思ったこと** - 自分の考えや意見を表現
- ✨ **自由テーマ** - 何でもOK、好きなことを書く

例えば、「今日の出来事」を選んだとします。

すると、ガイド質問が出てきます：

> **「昨日、何があった？」**
>
> あなたの回答（日本語）：「プレゼンテーション失敗。資料の誤字に気づかず、上司に指摘された。落ち込んだ。」

次の質問：

> **「どう感じた？」**
>
> あなたの回答：「ショックだったけど、もっと注意深くなる機会だと思う」

最後の質問：

> **「今、どう思ってる？」**
>
> あなたの回答：「次はしっかり確認してから提出しよう」

これで、入力完了です。**たったこれだけ。日本語で、合計3-4行です。**

### 4-2. AIが生成する5つの学習素材

あなたが「生成」ボタンを押すと、**AIがあなたの日本語を読んで、自動的に以下の5つを生成します**：

#### ① 英文ナラティブ（メイン）

```
Yesterday, I gave a presentation at work, but I made a careless mistake.
I didn't notice a typo in my slide until my supervisor pointed it out.
I felt shocked at first, but I'm starting to see it as an opportunity to be more careful.
Moving forward, I'll double-check everything before submitting.
```

自然で、あなたの文脈に沿った英文が生成されます。

#### ② キーフレーズ（最大5個）

```
🔹 "give a presentation"
   意味：プレゼンテーションをする
   使い方のヒント：会議や説明の場面で。単に "present" より自然

🔹 "careless mistake"
   意味：うっかりした誤り
   使い方のヒント：自分の責任による小さなミスに。"silly mistake" より丁寧

🔹 "point it out"
   意味：それを指摘する
   使い方のヒント：誰かが問題を見つけて教える時に

🔹 "see it as an opportunity"
   意味：それを機会として見る
   使い方のヒント：ネガティブなことをポジティブに捉える表現

🔹 "double-check"
   意味：二重確認する
   使い方のヒント：ミスを防ぐために、もう一度確認する
```

単に意味だけでなく、「どういう文脈で使うのか」「他の似た表現との違いは何か」という、実用的な情報が含まれています。

#### ③ 代替表現（最大2個）

```
💡 代替1: "I failed to notice..." instead of "I didn't notice"
   ニュアンス：より正式・文語的な言い方。ビジネスレター向け

💡 代替2: "My supervisor brought it to my attention"
   ニュアンス：さらに丁寧な表現。より改まった場面向け
```

同じ意味でも、シーンによって使い分けられる表現を提供します。

#### ④ リコールテスト（復習問題）

```
【日本語プロンプト】
「昨日のプレゼンで、資料の誤字に気づかなかった。
それをあなたの上司が指摘した。
このシーンを、英語で説明してみてください」

【あなたが試しに作文】
"Yesterday, I gave a presentation, but there was a typo in my slide.
My supervisor pointed it out."

【解答例（AIが生成）】
"Yesterday, I gave a presentation at work, but I made a careless mistake.
I didn't notice a typo in my slide until my supervisor pointed it out."
```

後日の復習時に、このテストが出題され、あなたが「英語で再現できるか」をチェックします。

#### ⑤ 発音ヒント（必要に応じて）

```
📢 "presentation" [ˌprezənˈteɪʃən]
   アクセント：プレゼンTEIション（第2音節）

📢 "careless" [ˈkeərləs]
   アクセント：CARE・less（第1音節）
```

難しい単語には、IPA記号とアクセント位置を記載。後で「正しく発音しながら復習する」ときに役立ちます。

### 4-3. 音声読み上げ（TTS）で発音練習

生成された英文を見た後、**あなたはクリック1つで、その英文を聞くことができます。**

さらに便利なことに：

- **文単位で再生可能** - 1文目だけ、2文目だけ、という単位で再生できる
- **速度調整可能** - 0.9倍速、通常速、1.1倍速から選べる
- **リピート再生** - 同じ文を何度も繰り返す

**使用例：**

朝、移動中に聞きながら、その英文を「追いながら言ってみる」（シャドーイング）ことができます。

```
AI音声：「Yesterday, I gave a presentation at work,」
あなた：「Yesterday, I gave a presentation at work,」（追いながら言う）

AI音声：「but I made a careless mistake.」
あなた：「but I made a careless mistake.」
```

このように、**耳と口を同時に使う**ことで、脳にさらに深く刻まれます。

### 4-4. SRS（間隔反復）で長期記憶化

さて、翌日。あなたのスマホに通知が来ます：

> 「『今日の出来事』の復習：今日は1件のエントリーがあります」

アプリを開くと、「復習」タブに、昨日書いたエントリーがリストアップされています。

あなたがそれをタップすると：

```
【リコールテスト画面】

「昨日のプレゼンで、資料の誤字に気づかなかった。
それをあなたの上司が指摘した。
このシーンを、英語で説明してみてください」
```

あなたが試しに頭の中で、または口に出して英語を作ってみます。

その後、「答えを見る」をタップすると：

```
【解答表示】

"Yesterday, I gave a presentation at work, but I made a careless mistake.
I didn't notice a typo in my slide until my supervisor pointed it out."
```

そして、重要なのが、**自己評価画面**です：

```
【あなたはこう評価する】

🟢 覚えていた（Easy）
🟡 かなり頑張って思い出した（Good）
🔴 うろ覚えだった（Hard）
⚫ 全く覚えていなかった（Forgot）
```

- 「覚えていた」と答えれば → 次の復習は7日後
- 「頑張って思い出した」と答えれば → 次の復習は3日後
- 「うろ覚え」なら → 次の復習は1日後
- 「忘れた」なら → 次の復習は翌日

**つまり、あなたの理解度に応じて、AIが最適な復習スケジュールを自動で調整します。**

この仕組みが、**Spaced Repetition（間隔反復）**の本質です。

### 4-5. 「年前の今日」機能で感動を呼ぶ

さらに、面白い機能があります。

カレンダーを開くと、過去の同じ日付のエントリーが見える。例えば：

```
【今年の12月27日に書いたエントリー】
プレゼンで失敗した話

【1年前の12月27日に書いたエントリー】
初めてのプレゼンに緊張していた話
```

1年前は「何も話せず、どう言ったらいいのかわからない」という悩みを抱えていたのに、

今年は「失敗したけど、工夫で次に活かそう」という成長が見えます。

**この「成長の実感」が、最高のモチベーション**になります。

### 4-6. 統計情報とストリーク

ダッシュボードには、こんな情報も表示されます：

```
📊 統計情報
━━━━━━━━━━━━━━━━━━━
✍️ 合計エントリー：127件
🔄 復習待ちの今日のエントリー：5件
⭐ 習熟済み（マスター）：34件
📅 連続書き込みストリーク：23日

🎯 今月の学習時間：約4時間32分
```

ストリーク（連続性）は、シンプルだけど強力なモチベーションの源です。

「23日連続」を途切れさせたくないから、「今日も書こう」という気持ちになる。

### 4-7. データ管理：エクスポート・検索

また、このアプリは、あなたのデータを囲い込みません。

```
【エクスポート機能】
✅ JSON形式でダウンロード
✅ CSV形式でダウンロード
✅ Markdown形式でダウンロード
```

いつでも、すべてのデータを取り出せます。

さらに、検索機能もあります：

```
【検索例】
「'presentation' という単語が出てくるエントリーを全部見せて」
→ プレゼンに関する過去のエントリーが一覧表示される
```

自分が学んだ表現を、テーマごとに整理し直すこともできます。

## セクション5：技術的なこだわり（開発ストーリー）

ここからは、一歩引いて、**「このアプリはどうやって実現されているのか」という技術的な背景**を紹介します。

技術に詳しくない読者には「へえ、そんな工夫が」という納得感を、技術好きな読者には「ちゃんと考えられてるな」という信頼感を与えるセクションです。

### 5-1. マルチモデル対応 + フォールバックで安定稼働

このアプリの核は、**生成AI** です。

ただし、単一のAIモデルに頼るのは危険です。なぜなら：

- 特定のサービスが停止することもある
- 突然、価格が値上がりすることもある
- モデルの性能にばらつきがあることもある

だから、このアプリは、**複数のAIモデルを段階的に使う**設計になっています：

```
【第1選択】Google Gemini 2.5 Flash
 ├─ 特徴：高速・高品質・コスト効率的
 ├─ 処理時間：1-2秒
 └─ 成功率：約95%

【第2選択】DeepSeek V3 / R1
 ├─ 特徴：論理的思考に強い
 ├─ 処理時間：2-3秒
 └─ 成功率：約90%

【第3選択】xAI Grok Beta
 ├─ 特徴：ユーモアと正確性のバランス
 ├─ 処理時間：2-3秒
 └─ 成功率：約88%
```

**もし Gemini が使えなくなっても、自動的に DeepSeek に、それでも駄目なら Grok に切り替わります。**

つまり、ユーザーは「このAIサービス、今日は調子悪いな」という煩わしさを感じることなく、いつも同じクオリティのサービスが受けられます。

### 5-2. 厳格な品質検証

ただし、AIが生成したものをそのまま使うわけではありません。


形式が違ったら、再生成させます。

```

例1：「This is a test. これはテストです。」→ 50%日本語 → ❌ NG
例2：「Tokyo is a big city.」→ 0%日本語 → ✅ OK
```

#### ④ 論理的一貫性チェック

そんな悔しさと、「これを解く方法があるはずだ」という信念の下、このアプリを開発しました。

今、このアプリを毎日使っているユーザーたちから、こんなメッセージをもらいます：

> 「2 ヶ月使ってみたら、実際の会話で英語が出てくるようになりました」
>
> 「フレーズ暗記より、ずっと定着しています」
>
> 「毎日 5 分だから、続けられます」

これらのメッセージが、開発を続ける最大の動機です。

**このアプリは、単なる商品ではなく、多くの「話せない中級者」の悩みを解くために作られた、ある種の「社会的課題解決ツール」です。**

あなたが、そのツールを使ってくださることで、あなたの英語が変わるだけでなく、このアプリもまた進化します。
